# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build Commands

**Desktop hub (no HSM):**
```bash
go build -o zennode hub_server.go tls_cert.go hsm.go hsm_noop.go
```

**Desktop hub with HSM support:**
```bash
./scripts/setup_softhsm.sh   # one-time SoftHSM2 initialization (token "zenwallet", PIN 1234)
go build -tags hsm -o zennode hub_server.go tls_cert.go hsm.go hsm_noop.go
```

**Mobile WASM binary:**
```bash
GOOS=js GOARCH=wasm go build -o static/main.wasm mobile_wasm.go
```

**Full demo (requires Anvil for local chain):**
```bash
./run_demo.sh          # standard mode
./run_demo.sh --hsm    # with HSM encryption
```

## Testing

There is no automated test suite. Testing is performed by running the full demo and exercising the keygen, passkey registration, and signing flows manually across desktop + two mobile browser tabs.

`test_wasm.go` exists only to verify tss-lib imports compile correctly under WASM.

## Architecture

ZenWallet is a **2-of-3 threshold MPC wallet** where cryptographic key material is never reconstructed on any single device. Three parties must be online for keygen; any two of three suffice for signing.

### Party Roles

| Role | Binary | Entry Point |
|------|--------|-------------|
| Desktop Hub | `zennode` (native Go) | `hub_server.go` |
| Mobile 1 / Mobile 2 | Browser WASM runtime | `mobile_wasm.go` → `static/main.wasm` |

**Desktop hub** (`hub_server.go`): HTTPS server on port 8081 that coordinates TSS ceremonies, handles message routing between parties, verifies WebAuthn assertions, optionally encrypts keyshares via HSM (PKCS#11), and broadcasts signed Ethereum transactions.

**Mobile WASM** (`mobile_wasm.go`): Compiled Go code running in mobile browsers. Exports JavaScript-callable functions (`wasmInit`, `wasmKeygen`, `wasmSign`, `wasmSetAuthToken`, etc.). Performs TSS signing using a keyshare decrypted in the browser via WebAuthn PRF.

**Frontend** (`static/index.html` + `static/keyshare_crypto.js`): Single-page app. Desktop view shows keygen controls and QR codes for mobile onboarding. Mobile view shows keygen participation, passkey registration, and signing. `keyshare_crypto.js` handles AES-256-GCM encrypt/decrypt using WebAuthn PRF-derived keys via HKDF-SHA256.

### Key Generation Flow

1. Desktop initiates DKG ceremony (`POST /keygen`)
2. All three parties exchange TSS wire messages via `/message` (POST) and `/poll` (GET)
3. Each party stores its own keyshare; mobile parties encrypt theirs in `localStorage` using a key derived from the WebAuthn PRF extension output (Secure Enclave-backed)
4. Desktop saves its keyshare to disk (`desktop_keys_<addr>.json`) or HSM-encrypted (`.enc`)

### Signing Flow

1. Mobile user taps "Sign" → WebAuthn `navigator.credentials.get()` triggers biometric prompt
2. Secure Enclave returns signed assertion + PRF output → JavaScript derives AES key → decrypts keyshare from `localStorage`
3. Decrypted keyshare passed to WASM via `wasmLoadDecryptedKeys`; auth token set via `wasmSetAuthToken`
4. WASM runs TSS signing ceremony with desktop hub acting as the second party
5. Desktop hub broadcasts the completed Ethereum transaction via its RPC client

### HSM Conditional Compilation

`hsm.go` (PKCS#11 integration) and `hsm_noop.go` (no-op fallback) use mutually exclusive build tags:
- `//go:build hsm` — real HSM via `github.com/ThalesGroup/crypto11`
- `//go:build !hsm` — no-op (default builds)

Both files implement the same `HSMManager` interface, so the rest of the code is build-tag agnostic.

### Important Design Constraints

- **WebAuthn requires HTTPS** — TLS is mandatory; self-signed cert is generated by `tls_cert.go` using local IP as SAN. Mobile users accept the cert warning once.
- **Passkey does not participate in MPC math** — it gates access to the locally stored keyshare only (decrypt → load into WASM → sign).
- **Auth tokens are single-use, 5-minute TTL** — `wasmClearAuthToken()` is called after each signing ceremony.
- **Ethereum defaults** — RPC at `http://127.0.0.1:8545`, chain ID 31337 (Anvil). Pass `-rpc` and `-chain` flags to target a real network.
- **Local IP detection** uses `ipconfig getifaddr en0`; change this if on a different interface.

## CLI Flags (zennode)

```
-port <int>          HTTPS port (default: 8081)
-id <string>         Party ID: "desktop", "mobile1", "mobile2"
-rpc <string>        Ethereum RPC URL
-chain <int64>       Ethereum chain ID
-hsm                 Enable PKCS#11 HSM encryption
-hsm-module <path>   PKCS#11 .so path (default: /usr/local/lib/softhsm/libsofthsm2.so)
-hsm-token <string>  HSM token label (default: "zenwallet")
-hsm-pin <string>    HSM PIN (interactive prompt if omitted)
```

## Key Dependencies

- **github.com/bnb-chain/tss-lib/v2** — threshold ECDSA (secp256k1); the core cryptographic engine
- **github.com/ethereum/go-ethereum** — Ethereum transaction construction and RPC broadcast
- **github.com/go-webauthn/webauthn** — WebAuthn/FIDO2 passkey registration and authentication
- **github.com/ThalesGroup/crypto11** — PKCS#11 HSM interface (HSM build only)
